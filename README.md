# Netty 网关学习项目

一个基于 Netty 从零实现的 API 网关项目。

这个项目的重点不是直接做生产可用网关，而是借助“网关”这个足够真实的场景，系统学习 Netty 的核心用法、设计方式和常见问题。

## 项目定位

这是一个学习型项目，目标是通过逐步实现一个简化版网关，掌握以下内容：

- Netty 服务端启动与连接管理
- `ChannelPipeline` 与 `ChannelHandler` 的职责划分
- HTTP 编解码与请求处理流程
- 请求转发与响应回写
- 异步编程模型与 `ChannelFuture`
- `ByteBuf` 生命周期与资源释放
- 超时、限流、连接池、日志、指标等网关基础能力

## 学习重点

通过这个项目重点理解：

- 一个请求如何在 Netty 的 pipeline 中流转
- 入站事件和出站事件的区别
- 服务端与客户端如何协同完成代理转发
- 如何正确处理异常、连接关闭和资源回收
- 如何在真实场景中拆分 handler、路由、过滤器和转发逻辑

## 当前已实现

当前版本已经具备一个最小可运行的 Netty HTTP 网关雏形：

- 启动 Netty HTTP 服务端，默认监听 `8080`
- 使用清晰的包结构组织启动、配置、服务端、路由、处理器代码
- 提供最小静态路由注册表
- 提供固定响应接口，便于后续逐步演进为真正的代理网关

已实现接口：

- `GET /`
- `GET /health`
- `GET /gateway/routes`

## 快速启动

编译：

```bash
mvn compile
```

运行：

```bash
mvn exec:java
```

启动后访问：

```bash
curl http://127.0.0.1:8080/
curl http://127.0.0.1:8080/health
curl http://127.0.0.1:8080/gateway/routes
```

## 当前包结构

当前先保持单模块实现，但包结构按后续可拆分的方式组织：

```text
org.gateway.nettygateway
├── bootstrap   启动入口
├── config      服务配置
├── handler     请求处理器
├── route       路由定义与注册表
└── server      Netty 服务端启动与初始化
```

这样后面即使拆成 `gateway-core`、`gateway-server` 等模块，也能平滑迁移。
